Adds a bkg component. Adds a bkg component named "_name" based on the object "_comp".

Parameters
_fracThis can ve a RooRealVar/RooFormulaVar or a double. The Roo object are set as yields of the current component.
In case it's a double a Roo object is created in the following way:
- _frac > 1 creates a RooRealVar with "_frac" value (in the fit the yield will start from "_frac" and float)
- |_frac| <= 1 created a RooFormulaVar connecting this background yield as a fraction "_frac" of the singla yield
- _frac < -1 same as _frac > 1 but the number is fixed. Meaning that the yield of this bacgkround will not float in the fit
optOptions: -> It is passed as options to getPdf()
- "-ibegin": Adds the bkg component in front and not at the back of the backgrounds list
myvars-> It is passed as argument of getPdf()

Returns a PDF starting from different kinds of objects This is one of the core function of the ModelBuilder. The template type "T" can be:
- RooAbsPdf -> in this case it simply returns itself
- string -> it used stringToPdf() to get a PDF from the string
- TTree -> it uses the RooKeysPdf class to extract a smooth pdf from the tree events distribution
- TH1F -> it used the RoohistPdf class to extract a PDF.

Parameters
_baseObject to start with to create the PDF
_nameName to give to the newly created PDF
myvarsThis works only if the given type is a string. While creating the model it looks if its parameters are already in "myvars". And if it finds them it uses them to create the new PDF. Does nothing if myvars contains no parameters or wrong parameters or if the given type is not "string".
weightIn case the starting object is a TTree distributions are created considering the variable "weight" as an event-by-event weight.
optOptions: This is important because every parameter name must be unique in RooFit.
"-c[]-v[]" (TTree only) : Using TTrees you can apply cuts on them before extracting shapes. The cuts go inside the "c" braces and in the v braces you must list comma separated all variables on which you cut. "-c" and "-v" always go together! e.g.: "-c[B0_PT > 50 && Kst_P > 200]-v[B0_PT,Kst_P]"
"-s(n)" (TTree only): n = 1 or n = 2 (default) defines how fine is the resolution of the smoothing

Parameters

optAll options available from GetFrame() plus:
